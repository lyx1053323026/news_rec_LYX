# Swing 协同过滤召回算法技术文档

## 一、算法概述

Swing 是一种基于用户-物品交互图的协同过滤算法，用于计算物品间的相似度。与传统的 ItemCF 不同，Swing 通过引入"用户对共现惩罚"机制，抑制高重叠度用户对带来的噪声信号，从而更准确地捕捉物品间的独立共现关系。

**核心思想：** 如果两个用户共同点击过大量相同的物品（高重叠度），则它们在一对新物品上的共现提供的"独立证据"较弱——这可能源于用户本身兴趣高度一致或异常行为，而非物品间的真实关联。

## 二、算法原理

### 2.1 数学公式

对于物品 $i$ 和 $j$，Swing 相似度定义为：

$$\text{sim}(i, j) = \sum_{u \in U_i \cap U_j} \sum_{\substack{v \in U_i \cap U_j \\ v \neq u}} \frac{1}{\alpha + |I_u \cap I_v|}$$

其中：
- $U_i$：点击过物品 $i$ 的用户集合
- $I_u$：用户 $u$ 点击过的物品集合
- $\alpha$：平滑参数（本实现中 $\alpha = 1$）
- $|I_u \cap I_v|$：用户 $u$ 和 $v$ 的物品重叠数

**权重衰减机制：** 用户对 $(u, v)$ 对 $\text{sim}(i,j)$ 的贡献与 `1 / (1 + 重叠物品数)` 成正比。重叠物品越多 → 权重越小 → 该用户对的共现信号被降权。

### 2.2 对比其他协同过滤算法

| 算法 | 相似度计算方式 | 特点 |
|------|---------------|------|
| **ItemCF** | 基于共现次数 + 位置权重 + 时间衰减 | 时序建模能力强，信号丰富 |
| **Swing** | 用户对重叠度惩罚 | 抑制同质化用户对，信号更独立 |
| **Word2Vec** | 向量余弦相似度 | 语义关联，冷启动友好 |

## 三、工程实现

### 3.1 计算优化策略

Swing 的原始计算复杂度为 $O(|I| \cdot |U|^2 \cdot |I_u|)$，对于大规模数据集不可行。本实现采用以下优化：

| 优化策略 | 参数 | 目的 |
|---------|------|------|
| 用户历史截断 | `MAX_ITEMS_PER_USER = 200` | 每个用户只保留最近 200 条点击，减少单用户计算量 |
| 物品用户采样 | `MAX_USERS_PER_ITEM = 300` | 热门物品随机采样至多 300 个用户，避免 $O(n^2)$ 爆炸 |
| Set 预计算 | `user_item_set` | 将用户列表转为 `set`，交集计算 $O(\min(|A|, |B|))$ |
| 多线程召回 | `multitasking` 库 | 召回阶段按用户分片并行处理 |

### 3.2 参数选择实验

实验数据：250,000 用户，34,688 物品，1,590,375 条交互记录（平均每用户 6.4 次点击，中位数 4 次）。

#### MAX_ITEMS_PER_USER 分析

| 阈值 | 超出用户数 | 占比 |
|------|:--------:|:---:|
| > 20 | 12,547 | 5.0% |
| > 50 | 1,341 | 0.5% |
| > 200 | 12 | 0.005% |

数据整体稀疏，`MAX_ITEMS_PER_USER = 200` 仅影响 12 个用户，实际约束力很弱。

#### MAX_USERS_PER_ITEM 消融实验

| 指标 | 200 | **300** | 500 |
|------|:---:|:-------:|:---:|
| HitRate@5 | 0.2810 | **0.2876** | 0.2889 |
| HitRate@10 | 0.3617 | **0.3634** | 0.3663 |
| HitRate@20 | 0.4365 | **0.4400** | 0.4392 |
| HitRate@50 | 0.5510 | **0.5525** | 0.5522 |
| 耗时 | ~3 min | **~3.5 min** | ~4 min |

**结论：** `MAX_USERS_PER_ITEM = 300` 是最佳平衡点。200→300 的 HR@5 提升约 0.66pp；300→500 几乎零提升，说明 300 个采样用户已充分覆盖有效共现信号。

### 3.3 召回阶段

对于每个目标用户，召回流程如下：

1. 取用户最近 2 次交互的物品作为种子
2. 对每个种子物品，从 Swing 相似矩阵中查询 top-100 相似物品
3. 应用位置衰减：`score = swing_sim × (0.7 ^ loc)`（越近的交互权重越高）
4. 合并去重，最终取 top-50 作为召回结果

## 四、代码结构

```
recall_swing.py
├── cal_sim(df)           # 计算物品-物品 Swing 相似矩阵
│   ├── 用户历史截断 (tail 200)
│   ├── 构建 user↔item 双向映射
│   ├── user_item_set 预计算（set 化加速交集）
│   ├── 热门物品用户采样 (max 300)
│   └── 三重循环计算 Swing 权重
├── recall()              # 多线程召回任务 (@multitasking.task)
│   ├── 种子物品选取（最近 2 次交互）
│   ├── Swing 相似度加权累加
│   └── 生成 label（验证模式匹配 ground truth）
└── __main__              # 主流程
    ├── 读取 click.pkl / query.pkl
    ├── 调用 cal_sim → 保存 swing_sim.pkl
    ├── 多线程召回 → 合并 → 排序
    ├── 计算 HR@K / MRR@K 指标
    └── 保存 recall_swing.pkl
```

**依赖关系：**
- 输入：`click.pkl`（用户历史点击）、`query.pkl`（验证/测试用户）
- 输出：`swing_sim.pkl`（相似矩阵）、`recall_swing.pkl`（召回结果）
- 融合权重：1.0（与 ItemCF、Word2Vec 同级）
