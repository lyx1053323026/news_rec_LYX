# 排序模型技术文档

## 一、整体架构

排序阶段承接多路召回结果，对每个 `<user, article>` 候选对构建特征，通过 LightGBM 分类器预测用户点击概率，按预测分降序排列得到最终推荐列表。

**流水线位置：**

```
recall.py（合并多路召回）→ rank_feature.py（特征工程）→ rank_lgb.py（模型排序）
```

**核心组件：**

| 模块 | 职责 |
|------|------|
| `rank_feature.py` | 特征工程：从原始数据构造 30+ 维排序特征 |
| `rank_lgb.py` | 模型训练与预测：LightGBM + GroupKFold 5 折交叉验证 |

## 二、特征工程（rank_feature.py）

### 2.1 特征全景

```
rank_feature.py
├── 文章属性特征（来自 articles.csv）
├── 用户历史交互统计特征
│   ├── 点击时间差统计
│   ├── 文章创建时间差统计
│   ├── 文章字数统计
│   └── 点击时段统计
├── 计数特征
│   ├── user_id_cnt          # 用户点击总次数
│   ├── article_id_cnt       # 文章被点击总次数
│   └── user_category_cnt    # 用户在该类目下的点击次数
└── 召回相似度特征（4 路）
    ├── itemcf_sim_sum / itemcf_sim_last
    ├── swing_sim_sum / swing_sim_last
    ├── w2v_sim_last / w2v_sim_sum_2
    └── binetwork_sim_last
```

### 2.2 文章属性特征

从 `articles.csv` 直接关联，包含：

| 特征 | 说明 |
|------|------|
| `category_id` | 文章类目 ID |
| `created_at_ts` | 文章创建时间戳（归一化到秒） |
| `created_at_datetime` | 文章创建时间的 datetime 格式 |
| `words_count` | 文章字数 |

### 2.3 用户历史交互统计特征

这些特征从用户的历史点击序列中统计得出，刻画用户的点击行为模式。

#### 2.3.1 时间间隔统计

| 特征 | 计算方式 | 含义 |
|------|---------|------|
| `user_id_click_article_created_at_ts_diff_mean` | `groupby(user_id)['created_at_ts'].diff().mean()` | 用户连续两次点击的文章创建时间差的均值 |
| `user_id_click_diff_mean` | `groupby(user_id)['click_timestamp'].diff().mean()` | 用户连续两次点击的时间间隔均值 |
| `user_click_timestamp_created_at_ts_diff_mean` | `(click_ts - created_at_ts).mean()` | 用户点击时间与文章创建时间的差值的均值 |
| `user_click_timestamp_created_at_ts_diff_std` | `(click_ts - created_at_ts).std()` | 上述差值的标准差，衡量用户点击新鲜度的波动 |

#### 2.3.2 点击时段统计

| 特征 | 计算方式 | 含义 |
|------|---------|------|
| `user_click_datetime_hour_std` | `click_datetime_hour.std()` | 用户点击时段的离散程度（24小时制） |

#### 2.3.3 文章内容统计

| 特征 | 计算方式 | 含义 |
|------|---------|------|
| `user_clicked_article_words_count_mean` | `words_count.mean()` | 用户历史点击文章字数的均值 |
| `user_click_last_article_words_count` | `words_count.iloc[-1]` | 用户最后点击文章的字数 |

#### 2.3.4 最近点击的锚点特征

| 特征 | 计算方式 | 含义 |
|------|---------|------|
| `user_click_last_article_created_time` | `created_at_ts.iloc[-1]` | 用户最后点击文章的创建时间 |
| `user_clicked_article_created_time_max` | `created_at_ts.max()` | 用户点击过的最新的文章创建时间 |
| `user_click_last_article_click_time` | `click_timestamp.iloc[-1]` | 用户最后一次点击的时间戳 |
| `user_clicked_article_click_time_mean` | `click_timestamp.mean()` | 用户平均点击时间 |

#### 2.3.5 候选-历史差异特征

将候选文章的属性与用户历史锚点做差，刻画候选与用户近期兴趣的偏差：

| 特征 | 公式 |
|------|------|
| `user_last_click_created_at_ts_diff` | `candidate.created_at_ts - user.last_article_created_time` |
| `user_last_click_timestamp_diff` | `candidate.created_at_ts - user.last_click_time` |
| `user_last_click_words_count_diff` | `candidate.words_count - user.last_article_words_count` |

### 2.4 计数特征

对核心维度做 GroupBy 计数，反应热度与活跃度：

| 特征 | 含义 |
|------|------|
| `user_id_cnt` | 用户总点击数 —— 体量用户活跃度 |
| `article_id_cnt` | 文章总被点击数 —— 体量文章热门度 |
| `user_id_category_id_cnt` | 用户在该类目下的点击数 —— 体量用户类目偏好强度 |

### 2.5 召回相似度特征

这些特征是排序阶段最核心的信号，直接复用多路召回的相似度矩阵，对每个 `<user, candidate_article>` 计算相似得分。

#### 2.5.1 ItemCF 相似度特征

基于 `itemcf_sim.pkl`（v2 策略计算的物品-物品相似矩阵）：

| 特征 | 计算函数 | 公式 / 说明 |
|------|---------|-------------|
| `user_clicked_article_itemcf_sim_sum` | `func_if_sum` | $\sum_{k=1}^{N} \text{sim}(I_k, A) \cdot 0.7^{\,loc_k}$ <br/>用户全部历史物品与候选 A 的加权相似度之和，越近的历史物品权重越高（$0.7^{loc}$，loc 从 0 起逆序） |
| `user_last_click_article_itemcf_sim` | `func_if_last` | $\text{sim}(I_{last}, A)$ <br/>仅用用户最后一次点击物品与候选的相似度，捕捉即时兴趣 |

#### 2.5.2 Swing 相似度特征

基于 `swing_sim.pkl`（Swing 算法计算的物品-物品相似矩阵）：

| 特征 | 计算函数 | 公式 / 说明 |
|------|---------|-------------|
| `user_clicked_article_swing_sim_sum` | `func_swing_sum` | 同 ItemCF sum 的加权方式，使用 Swing 相似矩阵，越近的历史物品权重越高 |
| `user_last_click_article_swing_sim` | `func_swing_last` | 仅用最后一次点击物品的 Swing 相似度 |

#### 2.5.3 Word2Vec 相似度特征

基于 `article_w2v.pkl`（Word2Vec 训练的文章向量）：

| 特征 | 计算函数 | 公式 / 说明 |
|------|---------|-------------|
| `user_last_click_article_w2v_sim` | `func_w2w_last_sim` | $\cos(\vec{v}_{last}, \vec{v}_{A})$ <br/>最后点击文章与候选文章的余弦相似度 |
| `user_click_article_w2w_sim_sum_2` | `func_w2w_sum(num=2)` | $\sum_{k=1}^{2} \cos(\vec{v}_{I_k}, \vec{v}_{A})$ <br/>最近 2 次点击文章各自与候选的余弦相似度之和 |

> **向量相似度定义：**
> $$\cos(\vec{a}, \vec{b}) = \frac{\vec{a} \cdot \vec{b}}{|\vec{a}| \cdot |\vec{b}|}$$

#### 2.5.4 Bipartite Network 相似度特征

基于 `binetwork_sim.pkl`（二部图协同过滤相似矩阵）：

| 特征 | 计算函数 | 公式 / 说明 |
|------|---------|-------------|
| `user_last_click_article_binetwork_sim` | `func_binetwork_sim_last` | 最后点击文章与候选的 Bipartite Network 相似度 |

## 三、模型训练（rank_lgb.py）

### 3.1 LightGBM 配置

```python
lgb.LGBMClassifier(
    num_leaves=64,          # 叶子节点数，控制模型复杂度
    max_depth=10,           # 最大深度，防止过拟合
    learning_rate=0.05,     # 学习率
    n_estimators=10000,     # 最大迭代轮数（配合 early_stopping 使用）
    subsample=0.8,          # 行采样比例
    feature_fraction=0.8,   # 列采样比例（特征 bagging）
    reg_alpha=0.5,          # L1 正则化
    reg_lambda=0.5,         # L2 正则化
)
```

**正则化策略（防过拟合）：**
- L1 + L2 双正则（`reg_alpha=0.5`, `reg_lambda=0.5`）
- 行/列双采样（`subsample=0.8`, `feature_fraction=0.8`）
- Early stopping：验证集 AUC 连续 100 轮不提升则停止
- GroupKFold：按 `user_id` 分组，确保同一用户不出现在训练集和验证集中

### 3.2 交叉验证策略

采用 **GroupKFold (n_splits=5)**：

```
          用户分组后的数据集
         /    |    \    \    \
      Fold1 Fold2 Fold3 Fold4 Fold5
      (训练/验证交替)
```

- 按 `user_id` 分组划分，避免用户级别的数据泄露
- 5 折交叉验证，每个样本都会被预测一次作为 OOF（Out-of-Fold）
- 测试集预测：5 折模型的概率预测取均值

### 3.3 训练流程

```
1. 读取 feature.pkl
2. 分离训练集（label 非空）和测试集（label 为空）
3. LabelEncoder 对 object 列编码
4. 5 折 GroupKFold 训练：
   ├── 每折训练 LGBMClassifier
   ├── 验证集 → OOF 预测
   ├── 测试集 → 5 折概率均值
   └── 保存模型 lgb{fold_id}.pkl
5. 特征重要性汇总（5 折均值）
6. 计算验证集 HR@K / MRR@K
7. 生成提交文件 result.csv
```

### 3.4 评估指标

排序阶段使用与召回阶段相同的评估指标：

| 指标 | 含义 |
|------|------|
| **HitRate@K** | 用户真实点击文章出现在推荐列表 top-K 中的比例 |
| **MRR@K** | 真实点击文章排名的倒数均值（排名越靠前越高） |

## 四、特征重要性

模型训练完成后输出各特征的重要性（Gain 增益），5 折取均值排序。通常召回相似度特征（尤其是 itemcf_sim_sum 和 w2v 系列）是最强信号，计数特征和统计特征提供补充。

## 五、代码结构

```
rank_feature.py
├── func_if_sum()              # ItemCF 加权求和特征
├── func_if_last()             # ItemCF 最后点击特征
├── func_swing_sum()           # Swing 加权求和特征
├── func_swing_last()          # Swing 最后点击特征
├── func_binetwork_sim_last()  # BipartiteNetwork 最后点击特征
├── consine_distance()         # 余弦相似度计算
├── func_w2w_sum()             # Word2Vec 加权求和特征（最近 N 次）
├── func_w2w_last_sim()        # Word2Vec 最后点击特征
└── __main__                   # 特征生成主流程
    ├── 加载 recall.pkl + click.pkl + articles.csv
    ├── 用户历史统计特征
    ├── 计数特征
    ├── 召回相似度特征（4 路）
    └── 保存 feature.pkl

rank_lgb.py
├── train_model(df_feature, df_query)   # 训练模式
│   ├── GroupKFold 5 折训练
│   ├── OOF 验证
│   ├── 特征重要性
│   └── 生成提交文件
├── online_predict(df_test)             # 在线预测模式
│   ├── 加载 5 折模型
│   └── 预测均值 → 提交文件
└── __main__
    ├── valid  → train_model()
    └── online → online_predict()
```

**依赖关系：**
- 输入：`recall.pkl`（多路召回合并结果）、`click.pkl`（用户历史点击）、`query.pkl`、`articles.csv`
- 依赖相似矩阵（由召回阶段生成）：`itemcf_sim.pkl`、`swing_sim.pkl`、`binetwork_sim.pkl`、`article_w2v.pkl`
- 输出：`feature.pkl`（特征矩阵）、`lgb{0-4}.pkl`（模型文件）、`result.csv`（提交文件）

## 六、使用方式

```bash
# 特征工程
python rank_feature.py --mode valid --logfile "feature.log"

# 模型训练（验证模式，含 OOF 评估）
python rank_lgb.py --mode valid --logfile "rank.log"

# 在线预测（生成最终提交结果）
python rank_lgb.py --mode online --logfile "rank_online.log"
```

在完整流水线中的位置（`test.sh`）：

```
recall.py → rank_feature.py → rank_lgb.py
```
